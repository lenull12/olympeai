# Page `/avocats` — Plan d'implémentation

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Créer une landing page métier complète `/avocats` pour OlympeAI, exhaustive en features, avec des démos visuelles de l'assistant en action (WhatsApp, email, logs).

**Architecture:** Nouvelle route Next.js `/avocats/page.tsx` + composants sections dédiés. Réutilise les composants UI existants (Reveal, LetterReveal). Chaque section feature a un mock visuel interactif (animation WhatsApp, email, logs, agenda). Stack : Next.js + Tailwind + Framer Motion, cohérent avec le site existant.

**Tech Stack:** Next.js App Router, Tailwind CSS, Framer Motion, Lucide Icons

---

## Structure de la page

```
1. Hero avocat              — headline spécifique + badge métier + CTA
2. Douleur chiffrée          — "Vous perdez Xh/semaine" (3 stats)
3. Features grid (4 blocs)   — chaque bloc = titre + texte + mock visuel animé
   3a. Gestion de dossiers   — mock: tableau de bord dossier (relances, échéances)
   3b. Recherche & Veille    — mock: email veille juridique animé
   3c. Communication client  — mock: conversation WhatsApp animée (qualif + RDV)
   3d. Productivité          — mock: dictée vocale → note + résumé email
4. Capacités détaillées      — liste exhaustive des features (icônes + texte, 2 colonnes)
5. Comment ça marche         — 3 étapes spécifiques avocat (mock WhatsApp par étape)
6. RGPD & Secret pro        — 3 cartes de garanties (Europe, isolement, purge)
7. Pricing teaser            — 3 paliers + CTA (reprend le composant Pricing existant ou version simplifiée)
8. FAQ avocat                — 5-6 questions spécifiques métier
9. CTA final                 — Offre Fondateur 5 cabinets + Cal.com
```

   → Rien à installer. On configure WhatsApp pour vous. Vous envoyez des messages, c'est tout.

3. **"Ça remplace un vrai secrétariat ?"**
   → Non, ça le complète. L'assistant gère les tâches répétitives. Votre secrétariat se concentre sur l'essentiel.

4. **"Et si l'assistant se trompe ?"**
   → Chaque action importante passe par votre validation. Il prépare, vous validez.

5. **"Quels types de dossiers ça gère ?"**
   → Droit de la famille, pénal, civil, commercial — l'assistant s'adapte à votre spécialité.

6. **"Je peux arrêter quand je veux ?"**
   → Oui. Sans engagement. Paiement mensuel. Vos données sont purgées à la demande.

## Risques & Tradeoffs

- **Volume de composants :** 11 nouveaux fichiers + 4 mocks. Risque de dette de maintenance. Mitigé par la structure par sous-dossier.
- **Mocks réalistes :** Les données fictives doivent être crédibles (noms français, numéros de dossier réalistes, jurisprudence plausible).
- **Performance :** Beaucoup de composants "use client" avec Framer Motion. S'assurer que le tree-shaking fonctionne et que les animations sont légères.
- **SEO :** La page `/avocats` doit avoir des meta tags dédiés et un contenu textuel suffisant (pas seulement des visuels).
